# Phase 1: Signal Engine Core — Indicators & Vote - Context

**Gathered:** 2026-04-20
**Status:** Ready for planning

<domain>
## Phase Boundary

Produce a pure-math indicator library and a LONG/SHORT/FLAT signal vote for any given OHLCV DataFrame. Zero I/O, zero network, fully fixture-tested to 1e-9 tolerance. Addresses requirements SIG-01 through SIG-08.

**In scope:**
- `compute_indicators(df)` — ATR(14) Wilder, ADX(20) Wilder with +DI/-DI, Mom1/Mom3/Mom12 (21/63/252-day returns), RVol(20) annualised
- `get_signal(df)` — 2-of-3 momentum vote gated by ADX>=25, returns int in {-1, 0, 1}
- `get_latest_indicators(df)` — scalar helper for downstream consumers
- Test suite with golden fixtures, dead-simple Python oracle, determinism snapshot

**Out of scope (belongs to later phases):**
- yfinance fetching (Phase 4)
- Position sizing, exits, pyramiding (Phase 2)
- State persistence (Phase 3)
- Any orchestration, CLI, email, dashboard, or scheduling

</domain>

<decisions>
## Implementation Decisions

### Golden fixture strategy

- **D-01:** Fixture data is a mix — one canonical ~400-bar real historical snapshot (committed CSV, source documented with date of pull) PLUS multiple small synthetic scenario fixtures (10–30 bars each) targeting every vote branch (ADX<25 FLAT, ADX>=25 LONG 3-vote, ADX>=25 LONG 2-vote, ADX>=25 SHORT 3-vote, ADX>=25 SHORT 2-vote, ADX>=25 split-vote FLAT, warm-up insufficient data). Scenario fixtures live alongside the canonical fixture.
- **D-02:** Golden values are generated by a dead-simple Python oracle committed under `tests/oracle/`. Oracle is pure loops (no pandas vectorization, no numpy tricks, no external TA library). Oracle is separate from the system under test so a bug in `signal_engine.py` cannot silently also appear in the oracle.
- **D-03:** Granularity: both. The 400-bar canonical fixture anchors full-pipeline regression. The small scenario fixtures surface per-branch failures by name (e.g., `test_adx_above_25_split_vote_returns_flat`).
- **D-04:** Regeneration is via a committed `tests/regenerate_goldens.py` script, run manually when formulas intentionally change. Never runs in CI. Git diff on golden CSV changes is the review surface.

### Module API shape

- **D-05:** Public surface of `signal_engine.py` is minimal: `compute_indicators(df) -> pd.DataFrame`, `get_signal(df) -> int`, `get_latest_indicators(df) -> dict`. Private helpers prefixed with underscore: `_atr`, `_adx`, `_plus_di`, `_minus_di`, `_mom`, `_rvol`. Phase 2 imports `compute_indicators` and `get_latest_indicators`; everything else is implementation detail.
- **D-06:** `get_signal(df)` returns a bare `int` in `{-1, 0, 1}` matching the `LONG=1`, `SHORT=-1`, `FLAT=0` constants locked in `CLAUDE.md`. No dataclass, no tuple — debug context is derivable from the DataFrame and is the caller's responsibility to surface.
- **D-07:** `compute_indicators(df)` returns a **new** DataFrame = input + indicator columns `[ATR, ADX, PDI, NDI, Mom1, Mom3, Mom12, RVol]`. Does not mutate the input. Matches the `SPEC.md` docstring (`"Adds ATR, ADX, PDI, NDI, Mom1, Mom3, Mom12, RVol columns"`) while keeping idempotence.
- **D-08:** `get_latest_indicators(df) -> dict` returns the last-row scalars: `{'atr': float, 'adx': float, 'pdi': float, 'ndi': float, 'mom1': float, 'mom3': float, 'mom12': float, 'rvol': float}`. Phase 2 sizing and the orchestrator consume this — `df.iloc[-1]` indexing stays inside `signal_engine.py`.

### Edge-case / NaN policy

- **D-09:** When the latest bar's ADX is NaN (warm-up <20 bars, insufficient data, or data pathology), `get_signal(df)` returns `FLAT` (0). Semantics: "no position taken." Matches the user-visible "no trade today" outcome and keeps the bare-int contract.
- **D-10:** NaN Mom values abstain from voting. Signal is FLAT unless at least 2 *non-NaN* moms vote the same direction. During warm-up (first 252 bars, Mom12 NaN) a 2-of-2 agreement on Mom1+Mom3 can still produce LONG/SHORT. No special warm-up branch — the rule falls out of "NaN abstains."
- **D-11:** Division-by-zero in +DI / -DI when `sum(TR)` over the 20-bar window is 0 (perfectly flat prices) returns **NaN** — let it propagate. Downstream ADX becomes NaN, signal becomes FLAT via D-09. No epsilons, no magic numbers.
- **D-12:** RVol when 20-day return std is exactly 0 returns **0.0** (mathematically correct). Phase 2's `vol_scale = clip(0.12 / RVol, 0.3, 2.0)` guard owns the sizing safety net (`SIZE-03`: "guard RVol ≤ 1e-9 as 2.0"). Phase 1 does not duplicate the guard.

### Test layout & determinism proof

- **D-13:** One test file: `tests/test_signal_engine.py`. Organized into classes: `TestIndicators` (per-indicator golden comparison), `TestVote` (per-scenario signal correctness), `TestEdgeCases` (NaN/div-by-zero policy), `TestDeterminism` (hash snapshot). Matches the roadmap success criterion verbatim (`pytest tests/test_signal_engine.py -k indicators_or_vote`).
- **D-14:** Determinism is proven by a committed hash-of-outputs snapshot at `tests/determinism/snapshot.json` storing SHA256 of each indicator series (ATR, ADX, PDI, NDI, Mom1, Mom3, Mom12, RVol) computed on the canonical 400-bar fixture. `test_determinism.py` recomputes and asserts hash equality. Any numpy/pandas upgrade that shifts float bits fails loudly.
- **D-15:** `pytest-freezer` is NOT required in Phase 1. Indicator math takes a DataFrame as input and never reads a clock. Save the decorator for orchestration phases (4+).
- **D-16:** Coverage goal is **named scenario fixtures covering every branch of the vote truth table**. No branch-coverage tooling (pytest-cov) for Phase 1 — named tests are clearer signal than percentage numbers. Required scenarios: `adx_below_25_flat`, `adx_above_25_long_3_votes`, `adx_above_25_long_2_votes`, `adx_above_25_short_3_votes`, `adx_above_25_short_2_votes`, `adx_above_25_split_vote_flat`, `warmup_nan_adx_flat`, `warmup_mom12_nan_two_mom_agreement`, `flat_prices_divide_by_zero`.

### Claude's Discretion

- Docstring convention (NumPy-style vs Google-style) — pick one and keep consistent
- Type hint coverage (public API only vs all internal helpers)
- Exact golden CSV format details (header row, float precision string, NaN representation) — decide during implementation, document in `tests/oracle/README.md`
- Internal helper organisation (single `_ewm_wilder` utility vs per-indicator private functions) — refactor as patterns emerge
- Whether `get_latest_indicators` lives in `signal_engine.py` or a tiny `signal_engine/adapters.py` if the file grows

</decisions>

<specifics>
## Specific Ideas

- **1e-9 tolerance is strict.** Oracle must be a dead-simple Python reference, not an external lib. Anchoring floating-point behavior to our own simple implementation (not Excel, not pandas-ta) avoids cross-tool divergence at that tolerance.
- **Wilder smoothing exactly as SIG-01 specifies:** `ewm(alpha=1/14, adjust=False, min_periods=14)` for ATR(14); analogous for ADX(20). `adjust=False` and `min_periods=period` are the non-obvious knobs — both are required.
- **Scenario fixture names double as documentation.** A failing test `test_adx_above_25_split_vote_returns_flat` tells the operator the truth-table row that broke, not just "signal_engine is broken."
- **CLAUDE.md forbids `datetime.now()` inside signal_engine.** Enforce structurally — the module has no `import datetime`. Test with a grep-style assertion or `ast` walk.
- **The 400-bar canonical fixture should be a real-instrument snapshot** (e.g., a dated yfinance pull of `^AXJO` daily bars). Commit the source-URL/command + pull-date in a README alongside the CSV so future regeneration is reproducible. Do NOT use a live yfinance call in any test.

</specifics>
