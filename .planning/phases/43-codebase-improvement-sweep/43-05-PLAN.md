<action>
1. At module level in `state_manager/__init__.py`:
   ```
   _MUTATE_STATE_ACTIVE = threading.local()
   ```

2. At the TOP of `mutate_state` (before any `fcntl.flock` call):
   ```
   prior = getattr(_MUTATE_STATE_ACTIVE, "value", False)
   if prior:
       raise RuntimeError(
           "mutate_state is not re-entrant — do not call mutate_state inside a mutate_state callback "
           "(caller: <use sys._getframe(1).f_code.co_name or inspect.stack to surface frame>)"
       )
   _MUTATE_STATE_ACTIVE.value = True
   try:
       # ... existing flock acquisition + callback invocation + save logic ...
   finally:
       _MUTATE_STATE_ACTIVE.value = prior
   ```
   - Flag is set BEFORE `fcntl.flock` so a nested call detects re-entry without ever attempting the lock (avoids the deadlock entirely).
   - `prior` is captured and restored (not hard-set to False) so future test wrappers or nested context managers compose correctly.
   - `finally` covers every exit path — normal return, callback exception, RuntimeError re-raise — guaranteeing the thread-local does not leak.

3. Tests in `tests/test_state_manager.py`:
   - `test_mutate_state_raises_on_reentrant_call`: callback that itself calls `mutate_state` → expect `RuntimeError` matching `r"not re-entrant"`.
   - `test_state_writable_after_reentrant_error`: trigger the RuntimeError above, then perform a normal `mutate_state` call from the same thread; assert it succeeds (proves finally cleaned up).
   - `test_mutate_state_clears_flag_on_callback_exception`: callback raises `ValueError`; assert flag is False afterwards and a subsequent `mutate_state` call succeeds.

4. Run full suite — confirm no regression.
