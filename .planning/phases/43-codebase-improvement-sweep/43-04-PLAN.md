<action>
Implement cache-first news read with atomic write semantics (D-04).

1. **Cache file location**: `.cache/news/<market>.json` under project root (create directory at import time with `pathlib.Path(__file__).parent.parent / ".cache" / "news"`; ensure `mkdir(parents=True, exist_ok=True)`). Do NOT use CWD-relative paths.

2. **Cache schema** (JSON-serialised `NewsResult`):
   ```
   {
     "items": [...],
     "error": null | "timeout" | ...,
     "fetched_at": "<ISO-8601>",
     "stale": false
   }
   ```

3. **Add `refresh_news_cache(market: str) -> NewsResult` in `news_fetcher.py`**:
   - Calls `fetch_news(market)` → `NewsResult`.
   - If `result.error is None` (success): write fresh result with `stale=False`.
   - If `result.error is not None` (failure): read existing cache; if present, REWRITE preserving its `items` and `fetched_at` but setting `error=result.error` and `stale=True` (last-good-data with error flag). If no prior cache exists, write `NewsResult(items=[], error=result.error, ...)` with `stale=False` so missing-cache state is preserved (not faked as stale).
   - Write using ATOMIC semantics:
     ```
     tmp_path = cache_path.with_suffix(".json.tmp")
     tmp_path.write_text(json.dumps(payload))
     os.replace(tmp_path, cache_path)   # atomic on POSIX
     ```
   - NEVER `open(cache_path, "w")` directly. NEVER `json.dump(fp)` to the live path.

4. **Add `load_news_cache(market: str) -> NewsResult` in `news_fetcher.py`**:
   - If cache file does not exist → return `NewsResult(items=[], error="cache_missing", fetched_at=epoch_0)`. (Distinct from stale: missing means cache has never been populated.)
   - If file exists but JSON parse fails → return `NewsResult(items=[], error="cache_corrupt", ...)`.
   - If file exists and parses → reconstruct `NewsResult` from payload. Performs NO HTTP.

5. **Update `dashboard_renderer/components/signals.py`**: replace the inline `fetch_news` call with `load_news_cache`. Render distinct UI states for "fresh clear", "stale clear (refresh failed at <ts>)", "news unavailable (no cache yet)", "blocked".

6. **Wire `refresh_news_cache` into `scheduler_driver.py`** as a job that runs after the daily OHLCV fetch for each market. Wrap each market's refresh in `try/except` so one market's failure does not abort the others; log structured warnings.

7. **Tests in `tests/test_news_fetcher.py`** — add:
   - `test_load_news_cache_missing_file_returns_cache_missing`
   - `test_load_news_cache_corrupt_file_returns_cache_corrupt`
   - `test_load_news_cache_stale_after_refresh_failure_preserves_items`
   - `test_refresh_news_cache_uses_atomic_write` — monkeypatch `os.replace` to assert it is invoked with `(tmp_path, final_path)` and that the tmp file is written before replace.
   - `test_concurrent_reader_never_sees_partial_json` — spawn a writer thread that writes a large payload repeatedly while a reader thread loads the cache N times; reader must never raise `JSONDecodeError`. (Atomic rename guarantees this.)
   - `test_dashboard_render_uses_cache_not_http` — monkeypatch `fetch_news` to raise; assert dashboard render succeeds and shows stale/missing UI.
</action>
