<action>
Implement fail-closed news gating per consensus review (D-02). Design contract resolved as follows:

1. **In `news_fetcher.py`**, add:
   ```
   @dataclass(frozen=True)
   class NewsResult:
       items: list[NewsItem]
       error: str | None
       fetched_at: datetime
   ```
   - All success paths return `NewsResult(items=[...], error=None, fetched_at=now)`.
   - Retry-exhaustion / network failure paths return `NewsResult(items=[], error="<typed reason>", fetched_at=now)` where reason is one of `"timeout"`, `"http_error"`, `"parse_error"`, `"network_unreachable"`.
   - NEVER return a bare list. NEVER raise out of `fetch_news`; convert all caught exceptions to `NewsResult` with `error` populated.

2. **In `news_filter.py`**, add:
   ```
   @dataclass(frozen=True)
   class CriticalEventResult:
       triggered: bool
       fetch_error: str | None
       gate_status: Literal["clear", "blocked", "unknown"]
   ```
   - Change `has_critical_event` signature to `has_critical_event(result: NewsResult) -> CriticalEventResult`.
   - Mapping:
     - `result.error is not None` → `CriticalEventResult(triggered=False, fetch_error=result.error, gate_status="unknown")`
     - `result.error is None` and a critical-event keyword matches → `(triggered=True, fetch_error=None, gate_status="blocked")`
     - `result.error is None` and no match → `(triggered=False, fetch_error=None, gate_status="clear")`

3. **In `daily_run.py`**, apply BLOCK_ON_FAILURE policy explicitly. Locate the news-gating call site; replace the existing boolean check with:
   - If `event.gate_status == "clear"`: proceed with signal generation.
   - If `event.gate_status in ("blocked", "unknown")`: SKIP signal generation for that market for that run. Log a structured warning including `event.fetch_error`. Record the skip in state so the dashboard can surface it.
   - Add a module-level constant comment: `# NEWS_FAIL_POLICY = "BLOCK_ON_FAILURE" — fetch failure treated as gate_status='unknown' and blocks signals (D-02)`.

4. **In `dashboard_renderer/components/signals.py`**, when rendering news context, display the `gate_status`. Render `"News: clear"`, `"News: BLOCKED — <reason>"`, or `"News: UNKNOWN — fetch failed (<error>)"`. Do not silently render as if clear.

5. **Audit all call sites** of `fetch_news` and `has_critical_event` (grep) and update each — listing them up front in the executor's task notes. No call site may still receive a bare list.

6. **Tests in `tests/test_news_fetcher.py`** — add the five cases (genuine no-news, critical event found, fetch failure timeout, fetch failure HTTP error, malformed response) AND a `daily_run` integration test that monkeypatches `fetch_news` to return `NewsResult(items=[], error="timeout", fetched_at=...)` and asserts no signals are emitted for that market.
</action>
