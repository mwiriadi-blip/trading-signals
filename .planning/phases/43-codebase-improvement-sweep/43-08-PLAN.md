<action>
1. **Create `dashboard_renderer/static/` directory** containing:
   - `dashboard.css` (extract all CSS string blocks from assets.py)
   - `dashboard.js`  (extract all JS string blocks)
   - Discrete HTML fragments as separate `*.html` files (one per logical block).

2. **Path resolution — MANDATORY pattern** in `dashboard_renderer/assets.py`:
   ```
   from pathlib import Path
   _STATIC_DIR = Path(__file__).resolve().parent / "static"
   _CSS = (_STATIC_DIR / "dashboard.css").read_text(encoding="utf-8")
   _JS  = (_STATIC_DIR / "dashboard.js").read_text(encoding="utf-8")
   ```
   - Do NOT use `os.getcwd()`, `"./static"`, or any relative string path.
   - Do NOT use `open("static/...")` without `Path(__file__).parent` anchoring.
   - `importlib.resources` is acceptable as an alternative if preferred; in that case use `importlib.resources.files("dashboard_renderer").joinpath("static/dashboard.css").read_text()`.
   - After extraction, `dashboard_renderer/assets.py` must be < 500 lines.

3. **Split `web/routes/dashboard/__init__.py`**:
   - `_cache.py` — cache state, TTL logic, refresh helpers
   - `_routes.py` — FastAPI route handlers
   - `_helpers.py` — HTML rendering helpers
   - `__init__.py` becomes a thin re-exporter (< 50 lines) that imports and re-exports public surface. Watch for circular imports (`_routes` may import from `_helpers` and `_cache`, but `_helpers` and `_cache` must not import from `_routes`).
   - Each new module < 300 lines.

4. **Verification commands** (run as part of executor's task, results gated in acceptance_criteria):
   - `wc -l dashboard_renderer/assets.py web/routes/dashboard/__init__.py web/routes/dashboard/_cache.py web/routes/dashboard/_routes.py web/routes/dashboard/_helpers.py`
   - From a tmp directory (`cd /tmp && python -c "from web.routes.dashboard import *"`) — proves path resolution does not depend on CWD.
   - `.venv/bin/pytest -x --tb=short`.
</action>
