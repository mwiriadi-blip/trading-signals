## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser cookie -> route layer | `tsi_trace_open` cookie crosses from untrusted client (operator browser) into the FastAPI route handler. Cookie is JS-owned (no HttpOnly), unsigned by design (D-12). |
| Route layer -> on-disk dashboard.html | Route layer reads dashboard.html bytes (rendered by main.py daily run via dashboard.render_dashboard) and applies per-request `replace()` substitution before serving. |
| Browser tap -> JS click handler -> cookie write | DOMContentLoaded handler writes `tsi_trace_open` cookie on `<details>` toggle. No server round-trip. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | Cookie injection via malformed `tsi_trace_open` (e.g. operator copy-pastes `tsi_trace_open=AAPL,javascript:alert(1)`) | mitigate | Allowlist filter at route layer (`_resolve_trace_open` keeps only keys in `_VALID_TRACE_INSTRUMENT_KEYS = {'SPI200', 'AUDUSD'}`). Pinned by Task 5 `test_tsi_trace_open_cookie_tampered_unknown_keys_filtered`. D-16 + RESEARCH §Security. |
| T-17-02 | Tampering | XSS via indicator scalar display (a state.json edit injects `<script>` into a scalar field) | mitigate | `html.escape(value, quote=True)` at every leaf interpolation in `_render_trace_inputs`, `_render_trace_indicators`, `_render_trace_vote`. Existing `tests/test_dashboard.py::TestSignalAsOfXss` pattern extended in `TestTracePanels::test_render_does_not_mutate_state` (deepcopy guard) + the per-leaf escape audit during code review. |
| T-17-03 | Tampering | XSS via `ohlc_window[*].date` strings (server-written but escape anyway for defence-in-depth) | mitigate | `html.escape(row['date'], quote=True)` in `_render_trace_inputs`. RESEARCH §Security explicit recommendation. |
| T-17-04 | Information Disclosure | Cookie `tsi_trace_open` leaks operator behaviour to a malicious LAN observer | accept | Cookie carries no secret, no PII, no instrument data — only UI preference (`SPI200`, `AUDUSD`). HTTPS-only droplet (existing v1.1 infrastructure) protects in transit. Adding `Secure` attribute on the JS cookie write per RESEARCH A4 = defence in depth. |
| T-17-05 | Denial of Service | Operator crafts a `tsi_trace_open` cookie with extreme length (1MB string) to exhaust route memory in the `.split(',')` parse | accept | FastAPI / Starlette default cookie-size cap is browser-enforced (~4KB). | T-17-06 | Spoofing | An attacker tricks the operator into clicking a magic-link-style URL with `?tsi_trace_open=...` query param hoping it's set as cookie | accept | Cookie is set by JS only on the `<details>` toggle event; no server-side `Set-Cookie: tsi_trace_open=...` is ever issued. | T-17-07 | Repudiation / Audit Gap | Cookie tampering attempts are NOT logged (operator can edit `tsi_trace_open` and the route silently drops bad keys) | accept | Single-operator tool. | T-17-08 | Elevation of Privilege | Cookie-driven `<details open>` state somehow grants access to gated content | accept | Cookie controls ONLY `<details open>` attribute on rendered HTML. The trace panels' content is rendered the same way regardless — `<details>` collapsed-vs-open is purely visual. No content gating. |
