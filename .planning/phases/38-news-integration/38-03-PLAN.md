| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-38-03-02 | Tampering / Elevation | SSRF via headline URL | mitigate | news_fetcher.py performs ZERO HTTP calls beyond yfinance.Ticker.news attribute access. AST gate test_no_server_side_url_prefetch enforces no requests.get/post/urllib/httpx call sites. |
| T-38-03-03 | DoS | Hung yfinance call | mitigate | _RETRY_EXCEPTIONS narrow-catches network errors with backoff_s sleep; HTTP_TIMEOUT_S applies through yfinance internals (per data_fetcher.py precedent) |
| T-38-03-04 | Information Disclosure | Cache file shared across users | accept | NEWS-01 explicit contract: "one fetch per market per day shared across users" — headlines themselves are public; per-user dismiss state is the privacy boundary (Plan 04) |
| T-38-03-05 | Tampering | Cache poisoning via rogue yfinance fields | mitigate | _normalise_item only persists typed NewsItem fields (title, url, publisher, pub_date, title_hash); unknown raw keys are discarded |
| T-38-03-06 | Tampering | Concurrent cache write corruption | mitigate | tempfile + os.replace atomic write — concurrent readers see either the old file or the new file, never a partial write |
| T-38-03-07 | Elevation | Cross-I/O-peer import of data_fetcher | mitigate | news_fetcher.py defines its own _get_yf(); AST test FORBIDDEN_MODULES_NEWS_FETCHER does NOT block 'data_fetcher' but RESEARCH §Open Q3 documents the discipline — reviewer responsibility (no test gate, but no value in importing data_fetcher either) |
