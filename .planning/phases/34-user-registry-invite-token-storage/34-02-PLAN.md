## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller -> mint_invite_token | invited_by_uid and email are caller-supplied; storage layer stores as-is (validation is Phase 37 route's job) |
| unhashed_token -> consume_and_create_user | Raw token from HTTP request (Phase 37); must never be persisted |
| flock window | Cross-process LOCK_EX on auth.json during consume; all mutation inside the window |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-02-01 | Elevation of Privilege | consume_and_create_user — token replay (use twice) | mitigate | consumed=True flip inside LOCK_EX window; hmac.compare_digest checks stored hash; second consume finds consumed=True and raises InviteAlreadyConsumed |
| T-34-02-02 | Information Disclosure | token comparison timing oracle | mitigate | hmac.compare_digest (constant-time) replaces direct string equality; raw token never logged |
| T-34-02-03 | Information Disclosure | raw token persisted in auth.json | mitigate | Only 'sha256:<hex>' stored; raw token never written to any file; grep gate in test_raw_token_not_in_auth_json enforces this |
| T-34-02-04 | Elevation of Privilege | expired invite accepted | mitigate | expires_at check inside flock window using datetime comparison; parsed and compared before marking consumed; raises InviteExpired (distinct from InviteAlreadyConsumed) |
| T-34-02-05 | Elevation of Privilege | disabled user still active | accept | disabled flag stored at Phase 34; enforcement happens at login layer (Phase 35); storage layer is a pure record |
| T-34-02-06 | Denial of Service | flock deadlock if save_auth called inside flock window | mitigate | consume_and_create_user uses _atomic_write_unlocked exclusively inside window; save_auth is NEVER called inside flock window (Pitfall 1) |
