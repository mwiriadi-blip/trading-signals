| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01-01 | Tampering | auth_store/_io.py load_auth v1->v2 migration | mitigate | Migration writes via save_auth() -> _atomic_write() (tempfile+fsync+os.replace), not in-place overwrite; corrupt-file quarantine preserved |
| T-34-01-02 | Information Disclosure | auth_store/_io.py — DEFAULT_AUTH_PATH in __init__ | accept | DEFAULT_AUTH_PATH is a filesystem path config; not a secret; low-value target |
| T-34-01-03 | Denial of Service | load_auth v1->v2 migration loop: recursive call if save_auth calls load_auth | mitigate | save_auth is a pure write (no reads); load_auth migration only calls save_auth, not load_auth — no recursion possible per design (Pitfall 4) |
