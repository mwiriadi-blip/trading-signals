  uuid: str           # uuid.uuid4().hex

`web/middleware/auth.py::_try_cookie` (Plan 01 baseline) currently validates `tsi_session` only. Plan 02 EXTENDS it to ALSO validate `tsi_trusted`:

UA→label heuristic (Claude's Discretion E+F):
```python
def _derive_device_label(user_agent: str, client_ip: str, granted_at_iso: str) -> str:
  ua = user_agent or ''
  # iPhone / iPad / iOS detection first
  if 'iPhone' in ua:
    if 'CriOS' in ua: name = 'iPhone Chrome'
    elif 'FxiOS' in ua: name = 'iPhone Firefox'
    else: name = 'iPhone Safari'
  elif 'iPad' in ua: name = 'iPad'
  elif 'Android' in ua: name = 'Android'
  elif 'Macintosh' in ua and 'Chrome' in ua: name = 'macOS Chrome'
  elif 'Macintosh' in ua and 'Firefox' in ua: name = 'macOS Firefox'
  elif 'Macintosh' in ua: name = 'macOS Safari'
  elif 'Windows' in ua and 'Chrome' in ua: name = 'Windows Chrome'
  elif 'Windows' in ua: name = 'Windows'
  elif 'Linux' in ua: name = 'Linux'
  else: name = 'Unknown device'
  # IP first-3-octets (RFC mask convention)
  parts = client_ip.split('.')
  ip3 = '.'.join(parts[:3]) + '.x' if len(parts) == 4 else client_ip
  date_str = granted_at_iso.split('T')[0]  # YYYY-MM-DD
  return f'{name} · {ip3} · {date_str}'
```
NOTE: This heuristic is good-enough for E-05 labels — operator-readable, uniqueness via (date, ip-prefix, browser) is fine for a single-operator device list. NOT used for security decisions.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator's previously-trusted device → AuthMiddleware | tsi_trusted cookie carries an itsdangerous-signed uuid; trusted via signature + auth.json revocation status |
| Operator's authenticated session → POST /devices/revoke | Cookie-session-only (NOT header) — denies remote-script revocation attacks (E-06 design choice) |
| auth.json `trusted_devices` array → on-disk | Atomic-write protection from Plan 01; revoked uuids retained for audit (write-only-once-revoked) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16.1-12 | Spoofing | tsi_trusted cookie replay after revocation | mitigate | Middleware checks `auth_store.is_uuid_active(uuid)` AFTER signature validation — revoked uuid is rejected even with valid signature; test asserts revoked tsi_trusted does NOT grant |
| T-16.1-13 | Tampering | UUID collision in trusted_devices on parallel issuance | accept | uuid4 collision probability ~10⁻³⁷; single-operator concurrent issuance impossible (one TOTP verify completes before next starts); collision = boot-time RuntimeError-tier, not user-facing |
| T-16.1-14 | Information Disclosure | /devices page leaks trusted-device labels (could include guessable IP first-octets) | accept | Single-operator tool; the operator IS the audience; IP first-3-octets is the operator's own home/mobile-carrier prefix, not sensitive |
| T-16.1-15 | Tampering | tsi_trusted cookie forgery via signature replay | mitigate | itsdangerous HMAC signature; secret = WEB_AUTH_SECRET (≥32 char per Phase 13 D-17 = ~128-bit entropy); same secret as tsi_session — salt `'tsi-trusted-cookie'` distinguishes purposes per RESEARCH §itsdangerous salt-non-optional |
| T-16.1-16 | Denial of Service | last_seen-update file-write storm degrades performance | accept | Single-operator droplet load N=1; auth.json <1KB; atomic-write <5ms. v1.2 optimization to in-memory dirty flag captured in deferred section |
| T-16.1-17 | Spoofing / CSRF | Remote attacker tricks operator into POST /devices/revoke-all | mitigate | SameSite=Strict on tsi_session cookie + form posts to same origin (browser blocks cross-site form submissions to /devices/* with the cookie); E-06 also requires cookie-session NOT header (defense in depth — header is curl-only, not browser-form-able) |
| T-16.1-18 | Privilege Escalation | Trusted-device cookie grants full dashboard access without TOTP — stolen device wins | accept | DOCUMENTED — operator opted into trusted-device per E-04 + E-08 (full-trust active session). Mitigation = operator revokes via /devices when device is lost. Recovery = magic-link reset (Plan 03) |
