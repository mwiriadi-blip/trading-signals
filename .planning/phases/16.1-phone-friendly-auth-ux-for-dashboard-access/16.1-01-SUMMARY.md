## Decisions Made

- **`is_cookie_session` 3-way semantics (None | True | False)** — chose sentinel-default rather than two boolean kwargs to keep the public render API one parameter wider, not two. Default `None` makes `main.run_daily_check` callers (no request context) emit placeholders for web-layer substitution; explicit True/False renders inline for direct/test callers.
- **Local serializer construction at register time** — built `URLSafeTimedSerializer` ONCE inside `register(app)` rather than lazily per-request. Constructor is non-trivial (HMAC key derivation); per-request `loads(token, max_age=43200)` is a pure signature check. Mirrors login.py / totp.py patterns.
- **Local import of dashboard helpers in web/routes/dashboard.py** — `_render_signout_button` and `_render_session_note` imported inside the request handler (not at module top). web/routes/dashboard is allowed to import dashboard per Phase 13 D-07; the LOCAL pattern aligns with Phase 11 C-2 import-only-when-needed convention.
- **Salt aligned to `tsi-session-cookie`** — RESEARCH §RESOLVED 2026-04-27 supersedes CONTEXT D-10 verbatim for grep-discoverability (cookie name root + salt root match).

- **Fix:** Added `pillow==11.3.0` to `requirements.txt` alongside `qrcode==8.2`. Plan artifacts already called for this; logged here for traceability.

- **Committed in:** `34b709a`

### 2. [Rule 3 — Blocking] /logout cookie-deletion attrs literal grep
- **Found during:** Task 3b
- **Issue:** Plan's verbatim grep expected ≥2 occurrences of the literal cookie-deletion attribute string in `web/routes/totp.py`. The actual implementation refactors them into a module-level `_COOKIE_ATTRS_DELETE` constant, so the literal substring appears once but is reused twice at runtime.
- **Fix:** Constant-based DRY approach kept; the runtime-effective behaviour is verified by the cookie-deletion-attrs-match-creation test rather than the literal grep. Documented in `2e5d9aa` commit message.
- **Verification:** `tests/test_web_routes_totp.py` cookie-deletion-shape tests pass.
- **Committed in:** `2e5d9aa`
