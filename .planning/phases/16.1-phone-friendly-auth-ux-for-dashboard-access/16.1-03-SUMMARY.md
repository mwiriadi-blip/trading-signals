# Phase 16.1 Plan 03: Magic-Link 2FA Reset + Rate Limits + Operator UAT Summary

Magic-link reset path (POST `/forgot-2fa` + GET `/reset-totp` + `?reset=1` enroll mode) for AUTH-11 lost-phone recovery, plus in-process rate limits across all auth endpoints (F-08), `OPERATOR_RECOVERY_EMAIL` boot validation (F-06), and the operator UAT runbook (`16.1-HUMAN-UAT.md`, 7 scenarios) — final wave of Phase 16.1 closing the cookie+TOTP+trusted-device+magic-link auth UX.

## What was built

### auth_store.py extension (Task 1)
4 new helpers for the F-01 PendingMagicLink schema:
- `add_magic_link(token_hash, action, expires_at, email=None)` — appends a 7-key row (token_hash + email defaulted from `OPERATOR_RECOVERY_EMAIL` + action + created_at + expires_at + consumed=False + consumed_at=None)
- `consume_magic_link(unhashed_token) -> tuple[bool, str | None]` — sha256-hashes the received token, finds the matching row, validates `consumed is False` AND unexpired, atomically flips `consumed=True` + `consumed_at=now-iso`, returns `(True, action)` on success
- `count_recent_magic_links(within_seconds=86400)` — used by F-08 per-account rate check
- `purge_expired_magic_links(retention_seconds=604800)` — drops rows older than 7-day audit window; called opportunistically from POST /forgot-2fa to keep auth.json bounded

### system_params.py constants (Task 1)
6 new constants — `MAGIC_LINK_TTL_SECONDS=3600`, `MAGIC_LINK_SALT='magic-link'`, `RATE_LIMIT_LOGIN_PER_15M=5`, `RATE_LIMIT_FORGOT_PER_HOUR=3`, `RATE_LIMIT_RESET_PER_HOUR=10`, `RATE_LIMIT_MAGIC_LINKS_PER_24H=3`. Middleware copy lives in `web/middleware/auth.py` (hex boundary).

### web/app.py — OPERATOR_RECOVERY_EMAIL boot validation (Task 1)
`_read_auth_credentials` now returns 3-tuple `(username, secret, recovery_email)`. Recovery email defaults to `mwiriadi@gmail.com`, validated against regex `^[^@]+@[^@]+\.[^@]+$`, fail-closed `RuntimeError` on malformed. Threaded onto `app.state.operator_recovery_email` for handlers.

### notifier.send_magic_link_email (Task 2)
Reuses Phase 12 `_post_to_resend` transport. Subject literal verbatim per F-03: `"Trading Signals — 2FA reset link (valid 1 hour)"`. HTML body has dark-theme inline-CSS button-link + plain-text fallback + AWST-formatted expiry timestamp. Failure modes (missing SIGNALS_EMAIL_FROM, missing RESEND_API_KEY, ResendError, unexpected Exception) all return `SendStatus(ok=False, reason=...)` — helper NEVER raises (CLAUDE.md "Email sends NEVER crash the workflow" preserved). 10 unit tests in TestSendMagicLinkEmail.

### web/middleware/auth.py — PUBLIC_PATHS + rate limits (Task 3)
- PUBLIC_PATHS extended: `frozenset({'/login', '/logout', '/enroll-totp', '/verify-totp', '/forgot-2fa', '/reset-totp'})`
- In-memory token-bucket via `_RATE_LIMIT_BUCKETS: dict[(ip, path) -> list[float]]`
- `_check_rate_limit` + `_get_client_ip` helpers (mirror `_log_failure` IP extraction)
- Dispatch checks rate limit BEFORE PUBLIC_PATHS check; over-budget → 429 (script) or 302 `/login?error=too_many_attempts` (browser navigation per `_is_browser_navigation`)
- Rate-limit constants live module-locally (cannot import system_params per hex boundary; drift risk accepted)

### web/routes/login.py — POST /forgot-2fa (Task 3)
- GET /forgot-2fa renders the reset form (mirrors `_render_login_form` shape; same `_LOGIN_INLINE_CSS`)
- POST /forgot-2fa: constant-time hmac.compare_digest validates BOTH username + password (T-16.1-21); per-account rate check via `count_recent_magic_links`; opportunistic `purge_expired_magic_links`; BASE_URL guard (no localhost fallback per LEARNING — missing → log ERROR + still render generic page); generates token via `URLSafeTimedSerializer(secret, salt='magic-link')`; persists `sha256(token)` via `add_magic_link`; sends email via `notifier.send_magic_link_email`. ALWAYS renders generic "Check your email" page regardless of cred validity, rate-limit hit, or send outcome (E-07 + T-16.1-21).

### web/routes/reset.py NEW (Task 3)
- GET /reset-totp validates signature + 1h max_age via itsdangerous; on `SignatureExpired` or `BadSignature` returns generic "Reset link is no longer valid" page (401)
- Calls `consume_magic_link(token)` to atomically flip `consumed=True`; on failure (unknown hash / already consumed / row expired) returns generic page
- On success: builds tsi_session token via `URLSafeTimedSerializer(secret, salt='tsi-session-cookie')`, 302 → `/enroll-totp?reset=1`, sets `Referrer-Policy: no-referrer` (T-16.1-25)
- All failure paths render the same single page — no leak of failure mode

### web/routes/totp.py — ?reset=1 mode (Task 3)
- GET /enroll-totp?reset=1 gates on tsi_session (just consumed magic-link), renders `_render_enroll_reset_choice_page` (Keep / New buttons)
- POST /enroll-totp with `action='keep'` → 302 / (no auth.json change)
- POST /enroll-totp with `action='new'` → regenerates totp_secret via `pyotp.random_base32()`, sets `totp_enrolled=False` via `auth_store.set_totp_secret`, renders fresh QR + issues new tsi_enroll cookie so the subsequent code-verify POST hits the Plan 01 baseline path
