## Accessibility Checklist

The login form is the most security-sensitive accessibility surface in the project. Every item below is mandatory; the executor MUST verify each before merging.

### Forms (`/login`)

- [ ] **Every `<input>` has a matching `<label for="...">`.** Required IDs: `login-username`, `login-password`. The `for` attribute on the label MUST exactly match the input's `id` attribute (browser screen-readers depend on this; pointer-clicking on a label focuses the input).
- [ ] **`autocomplete` attributes are set correctly:**
  - Username field: `autocomplete="username"` (NOT `"on"`, NOT omitted) ŌĆö iOS Safari + 1Password rely on this for keychain auto-fill.
  - Password field: `autocomplete="current-password"` (NOT `"new-password"`, NOT `"off"`).
- [ ] **Form-level `autocomplete="on"`** so iOS Safari offers to save the credentials after the first successful login.
- [ ] **`required` attribute on both inputs.** Browser shows native validation hint for empty submit (server still validates).
- [ ] **`autofocus` on the username input** so iPhone users can start typing immediately without an extra tap.
- [ ] **Tab order is exactly: Username ŌåÆ Password ŌåÆ Submit button.** No hidden inputs in between (the `?next=` value is in the form's `action` attribute, not a hidden input).
- [ ] **Error region uses `role="alert"` and `aria-live="polite"`** so screen-readers announce the "Sign in failed" message on the re-rendered page without needing focus management.
- [ ] **`<button type="submit">` not `<input type="submit">`** ŌĆö `<button>` is the accessible default and supports inner content semantics.
- [ ] **Color contrast (WCAG AA):**
  - `--color-text` `#e5e7eb` on `--color-bg` `#0f1117` ŌåÆ contrast ratio ~14.0 Ō£ō
  - `--color-text-muted` `#cbd5e1` on `--color-surface` `#161a24` ŌåÆ contrast ratio ~10.4 Ō£ō
  - `--color-text-dim` `#64748b` on `--color-bg` `#0f1117` ŌåÆ contrast ratio ~4.6 Ō£ō (just above 4.5 AA threshold for body text)
  - `--color-long` `#22c55e` on `--color-bg` `#0f1117` (button border + text) ŌåÆ contrast ratio ~7.0 Ō£ō
  - `--color-short` `#ef4444` on `--color-bg` `#0f1117` (Sign Out button) ŌåÆ contrast ratio ~5.1 Ō£ō

- **CSS extraction to `web/static/styles.css`** ŌĆö would require `app.mount('/static', ...)` in `web/app.py` AND updating Phase 13's middleware ordering to mount static BEFORE auth middleware (per global LEARNINGS "Static Assets Must Be Served Before Auth Middleware"). Defer to v1.2 if `_LOGIN_INLINE_CSS` duplicates more than ~50 lines of `_INLINE_CSS`. Today the duplication is the `:root` block + maybe 30 lines of form rules ŌĆö acceptable to inline.
- **Login form branding/logo** ŌĆö no SVG, no image asset. Text-only "TRADING SIGNALS" eyebrow is sufficient. Adding a logo requires a binary asset + a static-files mount = scope creep.
- **Password visibility toggle (eye icon)** ŌĆö extra JS for a single-operator tool that types one password. Defer.
- **Password strength meter** ŌĆö `WEB_AUTH_SECRET` is operator-set and Ōēź32 chars per Phase 13 D-17. No strength meter needed; the constraint is configuration-time, not entry-time.
- **Forgot-password / recovery flow** ŌĆö no recovery; operator owns the `.env` file on the droplet. Recovery = `ssh root@droplet vim /opt/trading-signals/.env`.
- **Multi-tab sign-out coordination** ŌĆö no service worker, no broadcast channel. Each tab has its own cookie state; closing tabs is the operator's responsibility per D-12 footnote.
- **CSRF token in login form** ŌĆö per CONTEXT D-13: `SameSite=Strict` cookie + single-form-per-origin = no CSRF surface for a single-operator tool. Skip.
- **Animation / transition on sign-in** ŌĆö no fade, no slide. Static page, instant nav.
- **Dark/light theme toggle** ŌĆö dashboard is dark-only by design; login/logout inherit. No toggle, no system-preference media query.
- **Login attempt rate-limiting UI** ŌĆö deferred per CONTEXT to a future nginx-layer change. The login form does NOT show "you have N attempts remaining" or similar. Failures all show the same generic "Sign in failed" string.

- **i18n / localization** ŌĆö single-operator, single-language (en-AU). No `lang` attribute changes needed beyond the existing `<html lang="en">`.
- **Logo, favicon, Apple touch icon** ŌĆö out of scope for this auth UX patch. Existing dashboard `<head>` stays unchanged.
