## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public Internet â†’ AuthMiddleware | Untrusted HTTP requests cross here; every byte (headers, query, body) is hostile until proven otherwise |
| Operator's `.env` â†’ create_app() | `WEB_AUTH_USERNAME` + `WEB_AUTH_SECRET` env vars cross from disk to in-memory at boot only |
| Browser address-bar `https://user:pw@host/` â†’ Authorization header | Operator-controlled credential passes through Safari Keychain; trusted in proportion to operator's password discipline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16.1-01 | Spoofing | `_try_basic` Basic Auth decode | mitigate | `hmac.compare_digest` for BOTH username and password (D-08 + AUTH-06); `base64.b64decode(..., validate=True)` rejects whitespace/non-alphabet; `partition(':')` unambiguous on no-colon malformed header â†’ return False |
| T-16.1-02 | Tampering | Authorization header injection | mitigate | UTF-8 decode wrapped in `try/except (binascii.Error, UnicodeDecodeError)` â†’ return False; no `int.from_bytes` or eval-equivalent |
| T-16.1-03 | Information Disclosure | 401 response leaking which path failed | mitigate | D-05 verbatim: 401 body is plain-text `unauthorized`, no `WWW-Authenticate` header, no path-specific hint; same response for missing-Basic, wrong-Basic, wrong-header, no-cookie |
| T-16.1-04 | Tampering | Sec-Fetch header spoofing to force 302 redirect | accept | An attacker could send `Sec-Fetch-Mode: navigate` to receive a 302 instead of 401, but the redirect target (`/login?next=â€¦`) is always the same internal page â€” no behavior change exploitable. The only "leak" is "Trading Signals has a /login page" which is also visible from the `?next=` query param echoing back |
| T-16.1-05 | Elevation | `==` comparison against WEB_AUTH_SECRET enabling timing attack | mitigate | `hmac.compare_digest` mandatory; AST guard test in `tests/test_web_auth_middleware.py::TestConstantTimeCompare` extended to include `_username_bytes` |
| T-16.1-06 | Denial of Service | WEB_AUTH_USERNAME containing `:` causing Basic Auth decode to misattribute the colon | mitigate | Boot-time RuntimeError at `_read_auth_credentials` if username contains `:` (D-08); systemd Restart=on-failure surfaces in journald; cannot reach runtime with bad config |
| T-16.1-07 | Repudiation | Audit log silently 3x-amplifying on multi-failed-auth requests | mitigate | Sampling pyramid 1: `_try_*` helpers return `bool` only, NEVER call `_log_failure`; only step 4 of dispatch logs; parametrized test asserts exactly 1 audit line per failed request |

All threats with `mitigate` disposition â†’ BLOCKING. T-16.1-04 explicitly accepted with rationale (no exploit surface beyond information that's already public via the redirect target).
