Admin nav link:
- Text: "Users"
- href: `/admin/users`
- Visible only when `role == "admin"` (server-side conditional render)
- Placement: in the existing admin dashboard nav, before or after existing links
  (executor determines exact position from existing nav structure)

## Interaction Contracts

### HTMX patterns (all mutations)

| Action | Method | Target | Swap |
|--------|--------|--------|------|
| Issue invite | `hx-post="/admin/invites"` | `#invite-form-wrapper` | `innerHTML` (shows success banner + URL) |
| Revoke invite | `hx-delete="/admin/invites/{token_hash}"` | closest `<tr>` | `outerHTML` (removes row) |
| Disable user | `hx-patch="/admin/users/{uid}/disable"` | closest `<tr>` | `outerHTML` (updates status cell) |
| Save email prefs | `hx-patch="/settings/email-prefs"` | `none` | `hx-swap="none"` + `handleTradesError` |

All admin mutations carry `hx-headers='{"X-Trading-Signals-Auth": "..."}'` (matching
existing dashboard pattern).

### Wizard navigation

Steps 1 ŌåÆ 2 ŌåÆ 3 ŌåÆ dashboard are full-page HTTP redirects (standard form POST ŌåÆ
302 redirect pattern). No HTMX partial swaps across wizard steps. Session-backed
state tracks which step the user is on (server-side, not client-side).

### Password input

- `type="password"`, `autocomplete="new-password"`, `minlength="12"`
- No client-side strength meter ŌĆö server validates length Ōēź 12 and returns 400
  with error block on failure
- Error display: `.error` region with `.error-heading` text
- Example error messages:
  - "Password must be at least 12 characters. Try again."
  - "Passwords do not match. Try again."

### Date picker

- `<input type="date">` native only ŌĆö no JS calendar library
- `min` attribute = today's ISO date (rendered server-side)
- Value sent as ISO string `YYYY-MM-DD`; empty string treated as `null` (no pause)

### Focus management

- First field `autofocus` on each wizard step (matches existing login/TOTP pattern)
- `focus-visible` outline: `2px solid var(--color-focus-ring)` at `outline-offset: 2px`
  (existing rule in `dashboard.html`, replicate in admin/invite CSS)

### Accessibility

- All error regions: `role="alert" aria-live="polite"`
- Success banner: `role="status" aria-live="polite"`
- Admin table: `<th scope="col">` on all column headers
- Disable/revoke buttons: `aria-label="Disable {email}"` / `aria-label="Revoke invite for {email}"` to disambiguate repeated button labels in table rows
- Step indicator paragraph is plain text (no ARIA needed ŌĆö not a progress indicator component)
- Touch targets: all row action buttons min-height 44px (via `--touch-target-min`)
