<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process env -> data_fetcher | IG_API_KEY, IG_USERNAME, IG_PASSWORD read from os.environ; never persisted; never logged in plaintext |
| data_fetcher -> IG REST API (HTTPS) | Outbound network call; credentials transit inside POST body over TLS; response tokens (CST, X-SECURITY-TOKEN) held in local dict for the fetch run only |
| IG response JSON -> _ig_normalise -> signal_engine | Untrusted external input; converted to DataFrame; downstream signal engine expects float64 numeric columns |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-02-01 | Information Disclosure | IG_API_KEY in log lines | mitigate | All log lines that reference api_key wrap it with `redact_secret(os.environ.get('IG_API_KEY',''))` (ASVS V2; pattern lifted from Phase 27 #13) |
| T-41-02-02 | Information Disclosure | IG_PASSWORD in log lines | mitigate | Password is NEVER logged — only used in the POST /session body; HTTP body is not logged anywhere in data_fetcher.py |
| T-41-02-03 | Information Disclosure | Session tokens (CST, X-SECURITY-TOKEN) in log lines or persisted files | mitigate | Session tokens live only in the in-memory `session_headers` dict during _fetch_via_ig; no log line interpolates them; no `save_state` call writes them; D-05 enforces no persistence |
| T-41-02-04 | Tampering / SSRF | IG_ACCOUNT_TYPE env var | mitigate | `_ig_base_url()` gates IG_ACCOUNT_TYPE.lower() against the fixed _IG_BASE_URLS dict {'demo','live'}; unknown values silently default to 'demo' — no attacker-controlled hostname can reach requests.post |
| T-41-02-05 | Denial of Service | Unbounded retry on transient errors | mitigate | Retry count capped at `retries` (default 3) with `backoff_s` between attempts; on exhaustion, return None and let yfinance fallback take over rather than block the daily run indefinitely |
| T-41-02-06 | Elevation of Privilege | Malicious operator forcing fallback via IG outage to inject yfinance-only data | accept | yfinance was the trusted source pre-Phase 41 and remains a documented fallback per D-01; an attacker who can block IG also cannot retroactively change the signal-engine contract; the dashboard warning surfaces fallback usage to the operator |
| T-41-02-07 | Tampering | Malformed IG response (e.g. missing snapshotTimeUTC, unexpected field types) crashing _ig_normalise | mitigate | _ig_normalise falls back to snapshotTime if snapshotTimeUTC absent (A4); _REQUIRED_COLUMNS contract check before return; any exception during normalisation propagates as DataFetchError which the caller handles |
</threat_model>
