<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Mac → GitHub (HTTPS) | git push over authenticated HTTPS or SSH; GitHub's auth is the trust anchor |
| GitHub → Droplet (deploy key) | INFRA-02 deploy key pulls authenticated commits to the droplet |
| Droplet localhost (127.0.0.1:8000) | Auth-gated by AUTH-01 shared-secret; SC-4 of Phase 11 keeps it bound to loopback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01-01 | Tampering | git push payload | accept | GitHub HTTPS/SSH provides integrity; deploy key on droplet pulls only from trusted origin |
| T-16-01-02 | Denial of Service | systemctl restart on droplet | mitigate | deploy.sh's curl retry loop (10 attempts @ 1s, REVIEWS HIGH #3) prevents premature failure declaration; uvicorn restart is local only |
| T-16-01-03 | Elevation of Privilege | sudo -n systemctl restart | accept | Sudoers entry per Phase 11 SETUP-DROPLET.md is scoped to the two unit names only (NOT NOPASSWD: ALL) |
| T-16-01-04 | Information Disclosure | smoke-check curl bypassing nginx | accept | curl runs on droplet localhost; WEB_AUTH_SECRET sourced from .env; no plaintext leakage off-droplet |
| T-16-01-05 | Repudiation | who deployed when | mitigate | Optional `[Deploy]` journald log line per CONTEXT.md §Specific Ideas; INFRA-02 nightly state-push commits also tag origin |
| T-16-01-06 | Spoofing | rogue push to origin/main | accept | Operator's GitHub credentials are the trust anchor; out of scope for Phase 16 |
</threat_model>
