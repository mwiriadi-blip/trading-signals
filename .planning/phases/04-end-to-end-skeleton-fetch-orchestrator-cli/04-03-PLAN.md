<test_strategy>
<!-- C-4 revision 2026-04-22 — caplog + basicConfig(force=True) strategy -->

**Problem:** `main.main()` calls `logging.basicConfig(level=logging.INFO, format='%(message)s', stream=sys.stderr, force=True)` at entry (Pitfall 4). The `force=True` detaches pytest's `caplog` handler from the root logger. Tests that call `main.main([...])` and then assert on `caplog.text` can see empty output or missing lines depending on test ordering and whether `caplog.set_level(...)` was called before or after the `main()` invocation.

**Chosen strategy (apply consistently across this plan AND 04-04-PLAN.md):** monkeypatch `main.logging.basicConfig` to a no-op in every test that asserts on `caplog.text` after calling `main.main(...)`. This preserves pytest's caplog handler attachment, lets the test set its own log level, and keeps the production code path identical (main() still calls the function, the no-op monkeypatch just swallows the call during tests).

```python
# Canonical fixture pattern for caplog-asserting tests:
def test_something_with_caplog_after_main(self, tmp_path, monkeypatch, caplog):
  caplog.set_level(logging.INFO)
  monkeypatch.chdir(tmp_path)
  monkeypatch.setattr('main.logging.basicConfig', lambda **kw: None)  # C-4 revision: preserve caplog handler
  state_manager.save_state(state_manager.reset_state(), path=tmp_path / 'state.json')
  monkeypatch.setattr(main.data_fetcher, 'fetch_ohlcv',
      lambda sym, **kw: _load_recorded_fixture(
        'axjo_400d.json' if sym == '^AXJO' else 'audusd_400d.json'))
  rc = main.main([...])
  # Now caplog.text contains the expected log lines because basicConfig was a no-op
  # and pytest's root-logger caplog handler survived.
  assert '[Sched] ...' in caplog.text
```

**Alternatives considered and rejected:**
- (a) Call `run_daily_check(args)` directly, bypassing `main()`. Rejected because CLI-04 and CLI-05 tests specifically want to exercise `main.main([...])` dispatch including the argparse path.
- (c) `caplog.set_level(logging.INFO, logger='root')` after main(). Rejected because by the time caplog's handler is re-attached, the log lines emitted during main()'s body have already been missed.

**When this is NOT needed:**
- Tests that do NOT assert on caplog (e.g. `test_closed_trade_to_record_gross_pnl_is_raw_price_delta` which only inspects return values).
- Tests that call `run_daily_check(args)` directly WITHOUT going through `main()` (e.g. `test_short_frame_raises_and_no_state_written` uses the direct-call form because Wave 3 hasn't yet wired the top-level exception boundary).

The `monkeypatch.setattr('main.logging.basicConfig', lambda **kw: None)` line is safe to include in every main()-invoking test as a defensive default — it adds ~1 line to each test and eliminates a whole class of ordering flakiness.
</test_strategy>
