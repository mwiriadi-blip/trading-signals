    ```python


    # =========================================================================
    # Phase 10 CHORE-02 / D-05: ruff F401 CI regression guard for notifier.py
    # =========================================================================

    def test_ruff_clean_notifier() -> None:
      '''CHORE-02 / D-05: notifier.py must have ZERO F401 (unused-import)
      warnings. Prevents regression of Phase 10 import cleanup.

      Invokes `ruff check notifier.py --output-format=json` via subprocess
      (ruff 0.6.9, pinned in requirements.txt). Parses the JSON array and
      asserts no entry has `code == 'F401'`.

      NOTE: This is the FIRST test in the codebase that shells out to a CLI
      tool and parses JSON output. Template per 10-RESEARCH.md §Pattern 2.
      Do NOT cargo-cult other patterns here — the subprocess+ruff idiom is
      stable across ruff 0.6.x and documented in ruff's JSON output schema.
      '''
      import json
      import subprocess

      result = subprocess.run(
        ['ruff', 'check', 'notifier.py', '--output-format=json'],
        capture_output=True,
        text=True,
        timeout=30,
      )
      # ruff exits 0 when clean, 1 when issues found. We do NOT assert on
      # returncode because the test goal is specifically "zero F401", not
      # "ruff exits 0" (future non-F401 warnings would erroneously red this).
      entries = json.loads(result.stdout) if result.stdout.strip() else []
      f401_entries = [e for e in entries if e.get('code') == 'F401']
      assert len(f401_entries) == 0, (
        f'CHORE-02: notifier.py must have zero F401 (unused-import) warnings; '
        f'found {len(f401_entries)}: '
        f'{[(e.get("location", {}).get("row"), e.get("message")) for e in f401_entries]}'
      )
    ```

    **Conventions:**
    - 2-space indent, single quotes (CLAUDE.md §Conventions)
    - Imports (`json`, `subprocess`) are LOCAL to the function body per
      project convention (matches many existing tests in test_main.py that
      do `import json` inside the test body)
    - Place AFTER the last existing class/function in tests/test_notifier.py
      — append to end of file (append-only, do not reorder)
    - No module-top imports added
    - `timeout=30` prevents hang; ruff is fast (~1s) so 30s is generous
    - `text=True` makes stdout str (simpler JSON parsing than bytes)
    - Assertion message shows (row, message) tuples for each offender —
      actionable for CI output
    - Do NOT pass `cwd=` — pytest runs from repo root; implicit cwd is correct

    **Subtle points per 10-RESEARCH §Executor notes:**
    - Assert ONLY on `code == 'F401'` (stable since ruff 0.1)
    - Do NOT assert on `fix.applicability` or `noqa_row` (version-variable)
    - Do NOT assert on `returncode` — other codes may legitimately exist
      later (e.g., E501 if future code goes long-line); the test scope is
