<objective>
Add the canonical `get_user_by_email(email, path=None) -> dict | None` helper to `auth_store/_users.py`, re-export it via `auth_store/__init__.py`, and lock its behaviour with unit tests. This is the prerequisite lookup primitive that the Phase 35 cookie-issue path (Plan 02) and `current_user` shim path (Plan 02) call to resolve a user_id from the username carried in the session payload.

Purpose: Without this helper there is no way to translate `payload['u']` (WEB_AUTH_USERNAME) into the `uid` field stored in `auth.json users[]` — the entire D-01..D-05 cookie-payload extension hinges on it being importable from `auth_store`.

Output: One new function + one re-export + one new test file (or extension if it already exists). No behaviour change to any caller in this plan.
</objective>
