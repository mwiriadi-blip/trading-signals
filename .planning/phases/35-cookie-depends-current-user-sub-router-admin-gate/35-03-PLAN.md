## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| middleware→dependency | request.state.user_id crosses from middleware-set context to handler-time dependency read |
| dependency→auth_store | uid string crosses into get_user lookup (one auth.json read per gated request) |
| dependency→client | 403 response body crosses back to client — JSON-only via locked detail strings (--reviews #5) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-03-01 | Elevation of privilege | require_admin role check | mitigate | Role read live from auth.json on every gated request via get_user(uid); cookie payload cannot smuggle role=admin because require_admin ignores any cookie-supplied role field. `test_admin_role_change_takes_effect_immediately` documents the live-read property. |
| T-35-03-02 | Spoofing | request.state.user_id forgery | mitigate | request.state is server-side only; middleware (Plan 02) is the only setter; cookie payload is itsdangerous-signed before it reaches state. |
| T-35-03-03 | Information disclosure | 403 detail string | mitigate | --reviews #5: detail strings are constants in source ('Not authenticated' / 'Admin access required'); neither reveals user existence or auth.json contents. Body is JSON only — no HTML stack trace leak. |
| T-35-03-04 | Tampering | getattr default vs AttributeError | mitigate | Use `getattr(request.state, 'user_id', None)` per PATTERNS.md to defend against PUBLIC_PATHS routes where the middleware never set the attribute — eliminates a 500-error attack surface. |
| T-35-03-05 | Denial of service | one auth.json read per gated request | accept | require_admin reads auth.json on every call — at single-admin scale this is fine; D-07 explicitly chose this over caching to keep role changes immediate. |
| T-35-03-06 | Tampering | hex boundary leak | mitigate | tests/test_web_healthz.py::TestWebHexBoundary scans the new file and fails the build if signal_engine/data_fetcher/main are imported. |
| T-35-03-07 | Information disclosure | 403 body leaking admin HTML content | mitigate | --reviews #5: FastAPI's default exception handler for HTTPException returns Content-Type: application/json with body {"detail":"<string>"} — Plan 05 asserts this end-to-end. No HTML or stack trace can leak. |
</threat_model>
