<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| middleware→dependency | request.state.user_id crosses from middleware-set context to handler-time dependency read |
| dependency→auth_store | uid string crosses into get_user lookup (one auth.json read per gated request) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-03-01 | Elevation of privilege | require_admin role check | mitigate | Role read live from auth.json on every gated request via get_user(uid); cookie payload cannot smuggle role=admin because require_admin ignores any cookie-supplied role field and uses the auth.json row. |
| T-35-03-02 | Spoofing | request.state.user_id forgery | mitigate | request.state is server-side only; not settable by client. Middleware (Plan 02) is the only setter; cookie payload is itsdangerous-signed before it reaches state. |
| T-35-03-03 | Information disclosure | 403 detail string | accept | Detail strings 'authentication required' / 'admin required' do not reveal user existence or auth.json contents — same string for both not-found and wrong-role per D-07. |
| T-35-03-04 | Tampering | getattr default vs AttributeError | mitigate | Use `getattr(request.state, 'user_id', None)` per PATTERNS.md to defend against PUBLIC_PATHS routes where the middleware never set the attribute — eliminates a 500-error attack surface on dependency-protected public routes. |
| T-35-03-05 | Denial of service | one auth.json read per gated request | accept | require_admin reads auth.json on every call — at single-admin scale this is fine; D-07 explicitly chose this over caching to keep role changes immediate. Re-evaluate at >100 admin-gated requests per second. |
| T-35-03-06 | Tampering | hex boundary leak | mitigate | web/dependencies.py forbidden imports asserted by tests/test_web_healthz.py::TestWebHexBoundary; AST walker scans the new file and fails the build if signal_engine/data_fetcher/main are imported. |
</threat_model>
