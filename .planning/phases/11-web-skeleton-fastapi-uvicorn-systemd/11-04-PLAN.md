    ## Install systemd unit

    Phase 11 commits the unit file to `systemd/trading-signals-web.service` in the repo. The operator copies it to `/etc/systemd/system/`, reloads the daemon, enables it for boot, and starts it.

    ```bash
    # Copy unit file from repo to system
    sudo cp /home/trader/trading-signals/systemd/trading-signals-web.service \
            /etc/systemd/system/trading-signals-web.service

    # Reload systemd to pick up the new unit
    sudo systemctl daemon-reload

    # Enable so it starts on boot (creates the multi-user.target.wants symlink)
    sudo systemctl enable trading-signals-web

    # Start now (does not wait for reboot)
    sudo systemctl start trading-signals-web

    # Verify it is active
    systemctl status trading-signals-web
    ```

    ## Install sudoers entry for trader

    `deploy.sh` calls `sudo systemctl restart trading-signals trading-signals-web`. To allow this without a password prompt (so the script is non-interactive and webhook-callable per INFRA-04), grant `trader` passwordless sudo for ONLY those two unit names.

    First, verify the actual `systemctl` path on the droplet (Ubuntu 24.04 LTS uses `/usr/bin/systemctl`; older systems may differ):

    ```bash
    which systemctl
    # Expected: /usr/bin/systemctl
    ```

    If the result is different, replace `/usr/bin/systemctl` with the actual path in the sudoers entry below.

    Create the sudoers file using `visudo` (NEVER edit `/etc/sudoers.d/` files with `nano` directly — `visudo` validates syntax before saving):

    ```bash
    sudo visudo -f /etc/sudoers.d/trading-signals-deploy
    ```

    Paste exactly this content (replace `/usr/bin/systemctl` if your `which systemctl` returned a different path):

    ```
    # Phase 11 D-21: trader may restart ONLY these two units, nothing else.
    # Used by deploy.sh `sudo systemctl restart trading-signals trading-signals-web`.
    trader ALL=(root) NOPASSWD: /usr/bin/systemctl restart trading-signals, /usr/bin/systemctl restart trading-signals-web
    ```

    Save and exit (`:wq` in vim, Ctrl+O / Ctrl+X in nano-as-visudo). visudo will refuse to save on syntax errors.

    Set permissions (sudoers.d files MUST be 0440 root:root or sudo silently ignores them on some distributions):

    ```bash
    sudo chmod 440 /etc/sudoers.d/trading-signals-deploy
    sudo chown root:root /etc/sudoers.d/trading-signals-deploy
    ```

    Validate the file independently (belt and suspenders):

    ```bash
    sudo visudo -c -f /etc/sudoers.d/trading-signals-deploy
    # Expected: /etc/sudoers.d/trading-signals-deploy: parsed OK
    ```

    Test that passwordless sudo works (this should NOT prompt for a password):

    ```bash
    sudo -n systemctl restart trading-signals-web
    # Expected: silent success. If it prints 'sudo: a password is required',
    # the sudoers entry has the wrong systemctl path — re-check `which systemctl`.
    ```

    Anti-pattern WARNING: NEVER grant `trader ALL=(root) NOPASSWD: ALL` or `trader ALL=(root) NOPASSWD: /usr/bin/systemctl *` — both create a privilege escalation path. The entry MUST list the two specific unit names per D-21.

    ## Verify port binding (WEB-02 / SC-4)

    The unit file binds uvicorn to `--host 127.0.0.1` (D-11). External attackers must NOT be able to reach port 8000 directly until Phase 12 nginx is in place. Verify with `ss`:

    ```bash
    ss -tlnp | grep 8000
    ```

    Expected output (the `127.0.0.1:8000` part is the critical assertion):

    ```
    LISTEN 0  ...  127.0.0.1:8000  0.0.0.0:*  users:(("uvicorn",pid=...,fd=...))
    ```

    If you see `0.0.0.0:8000` on the local-address column, the unit file is wrong — open a bug, fix the `--host` flag in `systemd/trading-signals-web.service`, re-copy, `daemon-reload`, restart. NEVER ship `0.0.0.0` in Phase 11.

    Also test the loopback HTTP path:

    ```bash
    curl -fsS http://127.0.0.1:8000/healthz
    # Expected: {"status":"ok","last_run":<YYYY-MM-DD or null>,"stale":<true|false>}
    ```

    From OUTSIDE the droplet (your laptop), confirm port 8000 is NOT externally reachable:

    ```bash
    # Replace <DROPLET_IP> with the droplet's public IP
    curl --max-time 5 http://<DROPLET_IP>:8000/healthz
    # Expected: connection refused or timeout (NOT a 200 response)
    ```

    If the external curl returns 200, the bind address is wrong — fix immediately.
