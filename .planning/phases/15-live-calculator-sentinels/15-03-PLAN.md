<objective>
Add `clear_warnings_by_source(state, source)` helper to state_manager.py. This is the surgical analogue to the existing `clear_warnings(state)` (Phase 8 D-02 full clear): it removes only warnings whose `source` matches the argument, leaving other-source warnings intact.

Why this exists: Plans 04 (main.py) and 06 (web/routes/trades.py) both need to "clear stale drift warnings then recompute" without nuking corruption/stale/sizing_engine warnings that other code paths own. The existing `clear_warnings` is too coarse (it empties the whole list).

The function is a pure dict operation — no fcntl, no atomic write, no I/O. Callers wrap it inside `mutate_state(_apply)` closures to get persistence atomicity. This preserves the v1.0 sole-writer invariant: `state_manager` remains the only module that mutates `state['warnings']`.

Purpose: Lock the partial-clear primitive that Plans 04 and 06 consume.
Output: One new function in state_manager.py + 5 populated test methods.
</objective>

    ```python
    def clear_warnings_by_source(state: dict, source: str) -> dict:
      '''Phase 15 D-02: filter out warnings whose `source` key matches the
      argument. Pure dict operation — no I/O. Caller wraps in mutate_state
      for persistence atomicity. Returns the same state dict for chaining.

      Preserves D-10 sole-writer invariant — state_manager is the ONLY
      module that mutates state['warnings']; notifier reads but never writes.

      Use case (Phase 15 SENTINEL-01..03): drift warnings lifecycle —
      clear_warnings_by_source(state, 'drift') then re-append fresh events
      from sizing_engine.detect_drift. Surgical, leaves corruption/stale/
      sizing_engine warnings intact.

      NOTE: This is DIFFERENT from clear_warnings(state) — that one wipes
      ALL warnings (used post-email-dispatch). Pitfall 8 in 15-RESEARCH.md.
      '''
      state['warnings'] = [
        w for w in state.get('warnings', [])
        if w.get('source') != source
      ]
      return state
    ```
