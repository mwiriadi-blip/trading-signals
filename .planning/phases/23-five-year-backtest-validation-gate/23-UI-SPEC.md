## Layout (page-level)

`/backtest` route layout, top to bottom:

1. **Page header** — `<h1>` "Backtest Validation" + subtitle `Strategy {STRATEGY_VERSION} — {N} years walk-forward (in-sample replay)`.
2. **Run summary banner** — pass/fail badge (large), combined cumulative return %, run timestamp (AWST), run metadata one-liner (`Initial $X AUD · SPI cost $Y · AUDUSD cost $Z`).
3. **Operator override form (D-14)** — three numeric inputs + `[Run with overrides]` button. Inline above tabs so operator sees current params before reading results.
4. **Tab strip** — three tabs: `Combined` (default), `SPI 200`, `AUD/USD`. ARIA `role="tablist"` per RESEARCH Pattern 3.
5. **Active tab panel** — equity curve chart (Chart.js canvas) + tab-local metrics row (6 metrics in `.stat-card` grid).
6. **Per-trade summary** — collapsible `<details>` with closed-trade table (last 20 visible by default, `Show all (N)` toggle). Reuses Phase 19 closed-trades table styles.
7. **History link** — footer link `View past runs →` to `/backtest?history=true`.

`/backtest?history=true` layout:

1. Page header `Backtest history`.
2. History table (D-06): version, run date, end date, cum return, pass/fail.
3. Overlay chart below table — one Chart.js line per run (cap 10 most recent), distinct colors, legend = run IDs.
4. Footer link `← Back to latest run`.

`/backtest?run=<filename>` reuses the `/backtest` layout (same render path), with the override form hidden (read-only historical view) and a `← Back to latest run` link in the header subtitle.

## Spacing Scale

Inherited from existing `dashboard.py:207-208` CSS custom properties — multiples of 4 only.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gap, inline label-value gap |
| `--space-2` | 8px | Compact element spacing, table cell padding-y |
| `--space-3` | 12px | Table cell padding-x, form input padding-y |
| `--space-4` | 16px | Default section gap, form row gap |
| `--space-6` | 24px | Card padding, section padding |
| `--space-8` | 32px | Inter-section breaks (tabs ↔ chart, chart ↔ metrics row) |
| `--space-12` | 48px | Page-level bottom padding |

Page wrapper: `max-width: 960px; margin: 0 auto; padding: 32px 24px 48px;` — same as existing dashboard wrapper.

Exceptions: tab buttons use a 44px min-height for touch targets on mobile (operator views from iPhone per Phase 16.1). Chart canvas: fixed `height: 320px` on desktop, `height: 240px` on viewports `< 600px`.

## Typography

Inherited from existing `dashboard.py:209` CSS custom properties. Three sizes + one display, two weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`--fs-body`) | 400 | 1.5 |
| Label | 12px (`--fs-label`) | 600 | 1.4 |
| Heading | 20px (`--fs-heading`) | 600 | 1.2 |
| Display | 28px (`--fs-display`) | 600 | 1.2 |

**Tabular numerics:** every numeric value (P&L, %, account balance, trade prices, dates) uses `font-family: var(--font-mono); font-variant-numeric: tabular-nums;` — matches dashboard convention, prevents column jitter.

**Pass/fail badge:** uses Display (28px / weight 600) for the `PASS` / `FAIL` word so it dominates the run summary banner.

## Color

Inherited verbatim from `system_params.py:132-140`. Dark surface aesthetic locked at v1.0; not relitigated.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) — bg | `#0f1117` | Page background |
| Secondary (30%) — surface | `#161a24` | Cards, table backgrounds, form input bg, tab panel bg |
| Border | `#252a36` | All `1px solid` separators (cards, tables, tabs underline) |
| Text | `#e5e7eb` | Default text color |
| Text muted | `#cbd5e1` | Labels, table headers, metadata |
| Text dim | `#64748b` | Empty-state body, footer copy |
| Accent — LONG / PASS (green) | `#22c55e` | PASS badge bg, equity-curve LINE for combined+per-instrument when net positive at end, table green P&L cells |
| Accent — SHORT / FAIL (red) | `#ef4444` | FAIL badge bg, drawdown shading, table red P&L cells |
| Accent — FLAT / pending (amber) | `#eab308` | Run-in-progress spinner ring (D-14 long-running POST), `APPROACHING` consistency with Phase 20 — NOT used on the backtest page itself but token reserved |

**60/30/10 split:**
- 60% — `#0f1117` page surface
- 30% — `#161a24` cards / form / chart container / table rows
- 10% — green `#22c55e` (PASS pathway) OR red `#ef4444` (FAIL pathway), never both at high prominence on the same page. Mutually exclusive on the run summary banner.

**Accent reserved for:**
1. Pass/fail badge background + border (the single dominant accent on the page).
2. Equity curve line stroke color (green if combined cum_return > 0 at end, red if < 0; intermediate drawdown periods are not colored — single line color per chart).
3. P&L cell text color in trade table (green if `net_pnl_aud > 0`, red if `< 0`, default text color if `== 0`).
4. History table `Pass/Fail` column cell text color.

Accent NEVER used for: tab buttons, form buttons, links, table borders, headings.

**Destructive:** none. The backtest UI has no destructive actions (no delete, no overwrite). FAIL state is informational red, not action-destructive red.

## Component contracts

### Pass/Fail badge

```
+----------------+
|  ✓  PASS       |
|  +127.45%      |
+----------------+
```

| Property | PASS | FAIL |
|----------|------|------|
| Background | `#22c55e` at 12% alpha | `#ef4444` at 12% alpha |
| Border | `1px solid #22c55e` | `1px solid #ef4444` |
| Word color | `#22c55e` | `#ef4444` |
| Glyph | `✓` | `✗` |
| Word | `PASS` | `FAIL` |
| Font | Display 28px / 600 / mono | same |
| Padding | `var(--space-6)` | same |
| Border radius | 8px | same |
| ARIA | `role="status"` `aria-label="Backtest result: PASS"` | `aria-label="Backtest result: FAIL"` |

Cumulative return % (display 28px, mono) renders below the word, same color as the word.

Placement: top of page, full-width inside the page wrapper, above the tab strip. Single instance per page (combined gating metric per D-16).

### Tabs

ARIA pattern from RESEARCH Pattern 3 — `role="tab"` / `role="tabpanel"`. URL hash drives active tab (`#tab=combined` default, `#tab=spi200`, `#tab=audusd`).

| Property | Inactive | Active |
|----------|----------|--------|
| Background | `#161a24` (surface) | `#0f1117` (bg — appears "raised" via inverted contrast) |
| Border-bottom | `1px solid #252a36` | `2px solid #e5e7eb` (text color underline) |
| Text color | `#cbd5e1` (muted) | `#e5e7eb` (text) |
| Font | Label 12px / 600 / uppercase | same + weight 600 |
| Padding | `var(--space-3) var(--space-4)` | same |
| Min-height | 44px | 44px (touch target) |
| `aria-selected` | `false` | `true` |

Tab panels use `hidden` HTML attribute (NOT `display: none` via class) per RESEARCH Pattern 3 — Chart.js needs the canvas to have nonzero size at instantiation, so all three are rendered on page load and only the wrapper visibility is toggled.

### Metrics row (per-tab)

Reuses existing `.stat-card` grid from `dashboard.py:265-298`. One row of six cards:

| Card | Label | Value format |
|------|-------|--------------|
| 1 | `Cumulative Return` | `+127.45%` (signed, 2dp, mono); color = green if `> 0`, red if `< 0`, default if `== 0` |
| 2 | `Sharpe (annualised)` | `0.84` (2dp, mono) — see naming note below |
| 3 | `Max Drawdown` | `-23.10%` (signed, 2dp, mono); always red text since DD is always ≤ 0 |
| 4 | `Win Rate` | `52%` (0dp percentage, mono) |
| 5 | `Expectancy` | `$142.30` (AUD, 2dp, mono); green if `> 0`, red if `< 0` |
| 6 | `Total Trades` | `178` (integer, mono) |

**Sharpe naming note:** JSON key is `sharpe_daily` per D-05 (locked); UI label says `Sharpe (annualised)` per RESEARCH Assumption A1 (the value is annualised `mean/std × √252`). The display label corrects the JSON key naming so the operator reads what the number actually is.

### Operator override form (D-14)

Three labelled numeric inputs in a single row (mobile: stacked), submit button at the end.

| Field | Label | Default | Min | Step | Validation |
|-------|-------|---------|-----|------|------------|
| `initial_account_aud` | `Initial account (AUD)` | `10000` | `0.01` | `100` | required, `> 0` |
| `cost_spi_aud` | `SPI 200 cost (AUD)` | `6.00` | `0` | `0.5` | required, `>= 0` |
| `cost_audusd_aud` | `AUD/USD cost (AUD)` | `5.00` | `0` | `0.5` | required, `>= 0` |

Submit button: `[Run with overrides]` — primary button styling (default text color on surface bg, `1px solid #252a36`, hover = lighter border).

**Long-running submit UX (D-14):**
- On `submit`, disable button + change label to `Running… (this can take up to 60s)`.
- CSS-only spinner ring (16px, amber `#eab308`, 1s rotation) appears next to the label.
- No JS framework dependency — inline `<script>` 8 LOC handles disable + label swap.
- Server returns 303 redirect to `/backtest` on success, 400 with inline error on validation failure (renders re-rendered form with `<p class="error">` above the field).

### History table (D-06)

| Column | Format | Sort |
|--------|--------|------|
| Strategy version | `v1.2.0` (mono) | n/a |
| Run date | `YYYY-MM-DD HH:MM AWST` (mono) | desc default |
| End date | `YYYY-MM-DD` (mono) | n/a |
| Cum return | `+127.45%` signed (mono); green/red text | n/a |
| Pass/Fail | `✓ PASS` (green) / `✗ FAIL` (red) | n/a |

Each row is a link (`<a>` wrapping `<tr>`'s logical row content) to `/backtest?run=<filename>`. Hover: row background lightens to `#1c2030` (surface +1 step).

### History overlay chart (D-06)

One Chart.js line chart, x = date, y = combined account balance AUD. Up to 10 lines (one per historical run). Legend renders run IDs (`v1.2.0-20260501T080000`). Line colors: Chart.js default palette cycles through; do NOT manually assign green/red here (would conflict with PASS/FAIL semantics).

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Page load with no `#tab=` hash | Combined tab active by default (D-04) |
| Page load with `#tab=spi200` hash | SPI 200 tab active; Combined panel hidden |
| Click tab | Active tab swaps; URL hash updates via `history.replaceState` (no page reload, no history pollution) |
| Submit override form (valid) | Button disables + spinner shows; server runs sync (~30-60s); 303 redirect to `/backtest` shows new run |
| Submit override form (invalid) | Server returns 400 with re-rendered form + inline error; button re-enables |
| Click history table row | Navigate to `/backtest?run=<filename>` (full page load, NOT pushState) |
| `?run=<unsafe path>` | Server returns 400; page renders error copy `Invalid backtest filename.` |
| Auth missing (browser GET) | 302 → `/login` (Phase 16.1 cookie middleware) |
| Auth missing (curl/non-browser GET) | 401 plain (Phase 16.1 D-04..D-07) |
| Auth missing (POST `/backtest/run`) | 401 plain |
| Tab switch with chart hidden | Chart instance NOT destroyed; `hidden` attr toggled (RESEARCH Pattern 3) |
| Mobile viewport `< 600px` | Tabs stay horizontal (3 tabs fit comfortably); metrics row wraps to 2 columns; chart height drops to 240px; override form stacks vertically |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | All text on bg/surface meets WCAG AA (4.5:1 body, 3:1 large) — inherited from existing dashboard, verified Phase 16.1 mobile UAT |
| Tab keyboard nav | Arrow keys move focus between tabs (left/right); Enter/Space activates; Home/End jump to first/last (RESEARCH Pattern 3 ARIA pattern) |
| Pass/fail badge | `role="status" aria-live="polite"` so screen readers announce on page load and on POST redirect |
| Chart canvas | `aria-label="Combined equity curve, $X to $Y over 5 years"` (one per canvas); `role="img"` |
| Form inputs | Each `<input>` has `<label for=...>`; required fields marked with `aria-required="true"`; validation errors linked via `aria-describedby` |
| Submit button disabled state | `aria-disabled="true"` during POST so screen readers announce the change |
| Tabular numerics | `font-variant-numeric: tabular-nums` prevents column jitter for screen-magnifier users |
| No motion-trigger seizures | Spinner uses 1s rotation only; no flashing |
