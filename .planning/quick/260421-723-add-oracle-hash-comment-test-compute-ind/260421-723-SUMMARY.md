## What was built

Three targeted polish artifacts landed on `main` as three separate atomic commits:

1. **`docs`** — Added an 18-line inline comment block inside `TestDeterminism::test_snapshot_hash_stable` explaining why the committed `snapshot.json` hashes oracle output (not production `compute_indicators`). Production has ~5e-14 drift vs oracle, inside the 1e-9 tolerance gate but not bit-identical. Cross-refs `01-REVIEWS.md §Top Finding` and `01-06-SUMMARY.md §Deviations #1`.

2. **`test`** — Added `TestIndicators::test_compute_indicators_is_idempotent` proving `compute_indicators(compute_indicators(df))` is bit-identical to a single call on the `axjo_400bar` canonical fixture. Uses `np.testing.assert_allclose(atol=0, rtol=0, equal_nan=True)` (strict bit equality, not 1e-9 tolerance). Also asserts column-order preservation and index identity.

3. **`feat`** — Added `tests/regenerate_scenarios.py`, an offline script that regenerates all 9 scenario fixtures from declarative `SEGMENT_RECIPES` mirroring `scenarios.README.md`. Does NOT import from `signal_engine.py` or `tests.oracle.*`. Runs deterministically (two invocations into two temp dirs produce byte-identical CSVs). Supports `SCENARIO_FIXTURES_DIR` env override for safe verification without touching committed fixtures.

- **Happy path:** test passes on first run → commit as `test(...)` → no production change.
